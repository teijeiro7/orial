## Design Context

### Users
Cristian, tracking his own life across gym, finance, and nutrition in one personal
dashboard. Used solo, on his own phone, in short glance-and-log sessions (checking
net worth, logging a set, logging a meal) — not a multi-tenant or public-facing product.

### Brand Personality
Premium, refined, technical, precise. Should feel like a well-crafted personal
finance/fitness instrument (Revolut/N26-style balance card is the explicit reference
already used for the Net Worth card) — not a generic admin dashboard, not playful.

### Aesthetic Direction
Dark theme (deep navy `#080C18` base), violet (`#7C3AED`/`#A78BFA`) as the primary
accent, cyan (`#06B6D4`) as a secondary accent. This palette is locked — it's the
established Orial identity carried through the Open Design HTML mockups
(`~/Library/Application Support/Open Design/.../*.html`) and the app's existing
`src/utils/colors.ts` token system. Do not propose replacing it wholesale, even
where it overlaps with generic "AI dark palette" tells — the fix is in execution
(depth, gradient richness, shadow, spacing, hierarchy), not in swapping hue.
Flat GlassCard variant (`src/components/GlassCard.tsx`) is the base surface;
the `glass` (blurred) variant exists but is used sparingly, not everywhere.

### Design Principles
1. Match the Open Design mockups' visual richness (multi-stop gradients, real
   shadows, generous radius) — the current RN implementation under-executes them
   (flat single-tone backgrounds, no shadow, smaller radius) even when the
   structure is correct.
2. Numbers are the hero. Typography hierarchy should make key figures (balance,
   totals, prices) unmistakably the loudest thing on any card.
3. Violet is the accent, not the wallpaper — reserve full-saturation violet for
   the 1-2 most important elements per screen (CTA, brand mark, primary metric).
4. No decorative-only elements (sparkline-for-vibes, glow-for-vibes). Every visual
   element ties to a real number pulled from the data layer — this codebase already
   has a strong "don't fabricate data" convention (see net-worth trend %, gym
   session duration) that extends to visual polish too: don't fake depth that isn't
   backed by an actual design decision.
5. Mobile-first single-column rhythm: vary spacing intentionally between sections
   (tight within a card, generous between sections) rather than uniform padding
   everywhere.
